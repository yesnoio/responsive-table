Step 1 — The Naive Approach

This approach doesn't work. All column headers stack at the top, completely disconnected from the data in each row.

A common first attempt at making a table responsive is applying display: block to <th> and <td> so they stack vertically on narrow screens:

table th,
table td {
  display: block;
}

Because the <thead> row renders first, all of its <th> cells pile up at the top — leaving data rows with no column context. Narrow this window to see the problem:

Column Foo Column Bar Column Baz
Alpha Beta Gamma
Delta Epsilon Zeta

The headers and data are disconnected — a reader has no way to know which value belongs to which column.
