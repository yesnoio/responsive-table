Step 2 — Add Labels to Each Cell

The table looks identical to a normal table at this stage — the labels are in the DOM but hidden. The next step reveals them on narrow screens.

Embed the column header text inside each <td> as a hidden .label span, and wrap the actual value in a .data span alongside it:

<style>
  table.responsive td .label { display: none; }
</style>

<table class="responsive">
  <thead>
    <tr>
      <th>Column Foo</th>
      <th>Column Bar</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <span class="label">Column Foo</span>
        <span class="data">Baz</span>
      </td>
      <td>
        <span class="label">Column Bar</span>
        <span class="data">Qux</span>
      </td>
    </tr>
  </tbody>
</table>

The table below has labels added. At any wide width it looks like a normal table — the labels are invisible but present in the markup:

Column Foo Column Bar Column Baz
Column FooAlpha Column BarBeta Column BazGamma
Column FooDelta Column BarEpsilon Column BazZeta

Now that every cell carries its own label, a media query can reveal them and stack the layout without losing column context.
