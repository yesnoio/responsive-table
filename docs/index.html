Responsive Tables
with Pure HTML & CSS

No JavaScript. No frameworks. A lightweight pattern that keeps column labels next to their data at any viewport width.

How It Works

The trick is to embed column header text directly inside each cell, hidden by default, then reveal it via a media query when cells stack.

✗

Naive Approach Fails

Stacking <th> and <td> with display:block isolates all headers at the top, disconnected from their data.

The Pattern

HTML

<table class="responsive">
  <thead>
    <tr>
      <th>Name</th>
      <th>Role</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <!-- Each td wraps a .label (column header copy) and .data -->
      <td><span class="label">Name</span><span class="data">Alice</span></td>
      <td><span class="label">Role</span><span class="data">Engineer</span></td>
    </tr>
  </tbody>
</table>

CSS

/* Labels are always in the DOM, hidden at wide widths */
table.responsive td .label {
  display: none;
}

@media screen and (max-width: 640px) {
  /* Hide the real header row */
  table.responsive thead {
    display: none;
  }

  /* Turn each row into a card */
  table.responsive tbody tr {
    display: block;
    border: 1px solid #ddd;
    border-radius: 6px;
    overflow: hidden;
    margin-bottom: 1rem;
  }

  /* Stack cells vertically; zero out td padding so label sits flush */
  table.responsive tbody td {
    display: block;
    padding: 0;
  }

  /* Remove bottom border on the last cell — the card border covers it */
  table.responsive tbody tr td:last-child {
    border-bottom: none;
  }

  /* Stack the spans inside each cell */
  table.responsive td span {
    display: block;
  }

  /* Style the label like a column header; owns its own padding now */
  table.responsive td .label {
    display: block;
    padding: 0.25rem 0.875rem;
    background-color: #ddd;
    font-weight: bold;
    text-align: center;
  }

  table.responsive td .data {
    padding: 0.6rem 0.875rem;
  }
}
